# Phase 6: Simulation Engine

This phase introduces the core automation feature of Presidio: The Simulation Engine. This allows the system to artificially generate hotel activity (bookings, check-ins, charges, housekeeping) over an accelerated timeline to demonstrate how the PMS responds under load.

## Proposed Changes

### 1. The Simulation Engine (Core Logic)
#### [NEW] [useSimulation.ts](file:///Users/lnaguit/Desktop/code/presidio/app/composables/useSimulation.ts)
A powerful Vue composable that acts as a timer-based state machine.
- **State:** Idle, Running, Paused.
- **Tick Interval:** Configurable speed (e.g., 1 "tick" every 2 seconds).
- **Event Weights:** Configurable probabilities for what happens on a tick (e.g., 40% chance of a new booking, 30% chance of a check-in, 20% chance of a housekeeping completion, 10% chance of a new charge on an open folio).
- **Execution:** Uses `setInterval` to trigger randomized actions against our Pinia stores, mimicking real-world staff and guest behavior.

### 2. Simulation Control Panel
#### [NEW] [simulation.vue](file:///Users/lnaguit/Desktop/code/presidio/app/pages/admin/simulation.vue)
A command center dashboard for the Admin to control the simulation.
- **Status Dashboard:** Shows current Simulation State (Running/Idle), Total Ticks, and Total Events Generated.
- **Transport Controls:** Large buttons for `Start`, `Pause`, `Step` (run exactly one tick manually), and `Reset`.
- **Configuration Sliders:** UI sliders allowing the user to adjust Event Weights in real-time. If you want a stress test of the Front Desk, crank up the "Check-In" weight!

### 3. Simulation Activity Feed
#### [NEW] [simulation-logs.vue](file:///Users/lnaguit/Desktop/code/presidio/app/pages/admin/simulation-logs.vue)
- Because we already built a global `useLogger` composable that tracks everything across modules, we will build a dedicated Activity Feed page here to stream the logs generated by the simulation in real-time.

## User Review Required

> [!WARNING]
> **Performance Consideration**
> The simulation will rapidly mutate localStorage via Pinia. By default, I will set the minimum tick speed to 2 seconds to avoid freezing the browser or maxing out Vue's reactivity limits on lower-end devices. Does this sound acceptable for the demo?
